# 📚 learning-hub
Repositorio central de la empresa para proyectos de aprendizaje y formación técnica, desarrollado como parte de los planes de desarrollo profesional del equipo.

Este repositorio reúne ejemplos, ejercicios y proyectos en distintos lenguajes y frameworks, con buenas prácticas de control de versiones, revisiones de código y colaboración.

## 🎯 Objetivo

- Centralizar los proyectos de formación del equipo

- Fomentar buenas prácticas de desarrollo colaborativo

- Estandarizar revisiones de código por tecnología

- Facilitar el crecimiento técnico individual y colectivo

## 🗂️ Estructura del repositorio
Los proyectos están organizados por **lenguaje o framework**:
| Carpeta | Revisores |
|--------|----------|
| /dotnet/** | Equipo .NET |
| /angular/** | Equipo Angular |
**Nota:** En caso de no encontrar una carpeta adecuada para su proyecto, solicitar a algún administrador del repositorio su creación. 

## 🔐 Flujo de trabajo
1. ❌ No se permite hacer push directo a main

2. ✅ Todo cambio debe realizarse mediante Pull Request

3. 🔍 Los PRs requieren revisión obligatoria según la carpeta modificada

4. ✅ Solo después de aprobación se permite el merge
## 🌱 Convención de ramas
Se recomienda usar el siguiente formato:
feature/tecnologia-descripcion

📄 Reglas generales

- El nombre del proyecto o solución debe tener el siguiente formato: **nombre_acronimo**

- Cada proyecto debe incluir su propio README.md

- Los cambios deben limitarse a la carpeta correspondiente

- El código debe compilar / ejecutar correctamente

- Mantener el repositorio organizado y limpio

🤝 Contribuciones

Este repositorio está pensado para uso interno del equipo.
Si tienes dudas sobre dónde ubicar un proyecto o quién debe revisarlo, consulta con el responsable de la tecnología.
